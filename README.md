# AI Drone Review Analyzer 🛸

Инструмент для автоматизированного интеллектуального анализа отзывов о дронах и квадрокоптерах с использованием современных языковых моделей (LLM). Система сравнивает результаты анализа от двух провайдеров: **Groq (Llama 3.1/Mixtral)** и **GigaChat (Sber)**.

## 📌 Описание проекта

Программа предназначена для обработки массива отзывов в формате JSON. Для каждого отзыва выполняется:
1.  **Определение тональности** (положительный, нейтральный, отрицательный).
2.  **Выделение главной темы** (например, "качество камеры", "время полета").
3.  **Поиск конкретных проблем** (issue extraction).
4.  **Предсказание рейтинга** на основе текста (1-5).

Скрипт сохраняет результаты в структурированные файлы для последующего анализа или импорта в BI-системы.

## ✨ Основные возможности

*   **Мультимодельный анализ:** Одновременная работа с Groq Cloud API и GigaChat API.
*   **Сравнение моделей:** Возможность сравнить, как разные нейросети интерпретируют один и тот же текст.
*   **Отказоустойчивость:** Автоматический перебор моделей внутри Groq (Llama 3.1 70B -> Mixtral -> Llama 3.1 8B) при сбоях.
*   **Обработка данных:** Автоматическая очистка ответов LLM от лишнего текста и парсинг "чистого" JSON.

## 🛠 Технологический стек

*   **Python 3.8+**
*   **Библиотеки:** `pandas`, `requests`, `openai`, `python-dotenv`
*   **LLM API:** 
    *   GigaChat API (SberDevices)
    *   Groq API (Llama 3.1, Mixtral)

## 🚀 Быстрый старт

### 1. Клонирование и установка зависимостей

```bash
git clone https://github.com/EshkereBOY/ai-drone-review-analyzer
cd ai-drone-review-analyzer
pip install -r requirements.txt
```

*Если файла requirements.txt нет, установите вручную:*
`pip install pandas requests openai python-dotenv urllib3`

### 2. Настройка окружения

Создайте файл `.env` в корневой директории и добавьте ваши API-ключи:

```env
# Groq API
GROQ_API_KEY=your_groq_api_key_here

# GigaChat API
GIGACHAT_CLIENT_ID=your_client_id_here
GIGACHAT_CLIENT_SECRET=your_client_secret_here
```

### 3. Подготовка данных

Подготовьте файл `reviews.json` в корне проекта. Формат входных данных:

```json
[
  {
    "text": "Отличный дрон, камера снимает супер, но батарея держит мало.",
    "rating": 4,
    "author": "Иван П.",
    "source_url": "https://..."
  }
]
```

### 4. Запуск анализа

```bash
python main.py
```

## 📊 Результаты

После завершения работы скрипта в папке `results/` появятся два файла:

1.  **`reviews.json`**: Полный файл, где к каждому исходному отзыву добавлены результаты анализа от каждой модели (`model_results`).
2.  **`detailed_results.json`**: Плоская структура данных, оптимизированная для анализа в Excel или Pandas (удобно для сравнения ответов Groq и GigaChat построчно).

## 📝 Пример структуры ответа (JSON)

Модели настроены возвращать строго определенную структуру:

```json
{
    "sentiment": "отрицательный",
    "main_topic": "время работы батареи",
    "issue": "аккумулятор держит заряд всего 5 минут",
    "rating": 1
}
```

## ⚠️ Примечание по безопасности

В коде реализовано подавление предупреждений SSL (`InsecureRequestWarning`), что необходимо для корректной работы с API GigaChat в некоторых корпоративных сетях. Используйте `verify=True` в методах `requests.post`, если работаете в доверенной среде.
